Как проходит урок

Не смотришь DevOps со стороны, а делаешь руками

В каждом уроке теория, терминал, задание и проверка находятся рядом. Формат похож на рабочую смену инженера: прочитал контекст, починил сервис, подтвердил результат.

  • теория короткая и привязана к задаче;
  • команды выполняются в изолированной учебной среде;
  • проверка смотрит конфиги, процессы, сеть и ответ сервиса.
Шаг 04

Nginx как reverse proxy

Настрой upstream для приложения и проверь, что внешний запрос получает HTTP 200.

  1. Открой конфиг в /etc/nginx/sites-enabled/app.conf
  2. Добавь proxy_pass на порт приложения
  3. Перезагрузи сервис и запусти проверку
$ sudo nginx -t
syntax is ok
test is successful

$ sudo systemctl reload nginx
$ rootops check nginx-proxy

FAQ

Коротко о формате

Это просто курс?

Нет. Курс здесь только часть продукта: главный смысл в учебной среде, практических заданиях и проверке результата.

Нужно ставить Linux, Docker и Kubernetes локально?

Нет. Идея RootOPS в том, чтобы запускать лаборатории в браузере и не тратить вечер на подготовку окружения.

Можно начать с нуля?

Да. Маршрут начинается с Linux, терминала и сетевой базы, а дальше постепенно ведет к контейнерам, CI/CD и Kubernetes.
