Delivery flow

От коммита до работающего сервиса

RootOPS показывает DevOps не как набор названий, а как цепочку действий: изменение, проверка, доставка, наблюдение и улучшение.

git push

Source

Код, контент, конфигурация и история изменений в одном репозитории.

actions

Validate

Проверка структуры, сборка артефактов и контроль секретов перед релизом.

rsync

Deploy

Доставка на VPS через SSH в директорию, которую раздаёт Caddy.

observe

Operate

Следующий слой: логи, метрики, алерты, backup и rollback-процедуры.

Runbooks and labs

Практика выглядит как рабочие задачи.

Каждая лаба должна давать результат, который можно проверить: сервис поднят, порт открыт, pipeline прошёл, метрика появилась, rollback отработал.

runbook 01

Static deploy через Caddy

Домен, HTTPS, firewall, директория сайта и первый автоматический релиз.

runbook 02

Dockerized app

Сборка образа, Compose, переменные окружения, healthcheck и restart policy.

runbook 03

Pipeline с rollback

GitHub Actions, SSH deploy, проверка результата и быстрый откат релиза.
